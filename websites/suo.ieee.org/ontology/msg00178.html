Dear Dietrich,
 
The base language of the STEP community is EXPRESS (ISO10303-11). This is an
entity-relationship language with both a lexical and (partial) graphical
form. It is able to express complex relationships and constraints. The
latest version currently being developed has as an explicit data type a
"connotational" subtype, which is explicitly for subtypes that can be
derived from other entity types.
 
Some of us feel that the entity relationship paradigm has some basic
limitations in having two layers, the data model layer, and the data layer
(there is of course also a meta-model layer - but that is not important
here). So we have started to develop a language called EXIST (EXpression of
Information based on logic and Set Theory). You can find some informal
information on EXIST at www.iso18876.org <http://www.iso18876.org>  if you
follow the link to the Preliminary Work Item (PWI) and look for the
documents at the bottom of the page.
