Greetings, Pat.  Way back when, you wrote:
> >;; Let REL be any n-place predicate, and let M be the numeral for the
> >;; positive integer m, where m is less than or equal to n.  Then we
> >;; let
> >;;
> >;;   (nth-domain REL M CLASS)
> >;;
> >;; serve as an abbreviation for
> >;;
> >;;   (forall (VAR_1 ... VAR_n)
> >;;           (=> (REL VAR_1 ... VAR_n)
> >;;               (instance-of VAR_m CLASS)))
> >
> 
> I should have noticed this sooner, but this doesnt really capture the 
> intended meaning of 'nth-domain', at least I don't think it does. 

It does.

> I assume that (nth-domain REL M CLASS) is supposed to mean that the
> Mth argument position of REL is restricted to be in CLASS, right?  

No.  What you've expressed is a metalinguistic syntactic restriction,
viz., that any *term* that occupies the nth argument position of the
*predicate* REL must be of the *syntactic* sort associated with the
class term CLASS.  If you want an nth-domain sentence '(nth-domain REL
M CLASS)' to say *that*, then the 'nth-domain' predicate should take,
not a predicate and a class term for its first and third argument
places, respectively, but a *name* for a predicate and a *name* for a
class term, respectively (e.g., quotation names formed in the usual
way).  

The language for which the above was designed is syntactically
unsorted, so such a restriction would be out of place.  To the
contrary, an nth-domain sentence says something about the *relation* R
that the predicate REL expresses, namely, that, in any instance
<a_1,...,a_n> of R, a_m must be a member of the class denoted by
CLASS.  It is thus an object language assertion.  To express it
directly one would have to allow predicates to serve as arguments to
other predicates, and that was not the case in the language in
question -- although it appears it will be allowed in SUO-KIF.  Hence,
nth-domain sentences are, strictly speaking, syntactically ill-formed
in that language, but the above gives us a way of using them all the
same -- understood, however, as abbrevations.  But if predicates *are*
allowed to be arguments to other predicates, then 'nth-domain' could
be considered just another predicate, and the abbrevational schema
above can be replaced by an *axiom* schema, viz.:

  For all predicates REL, numerals M, class terms CLASS, and variables
  VAR_1, ..., VAR_n,

  (=> (nth-domain REL M CLASS)
      (forall (VAR_1 ... VAR_n)
              (=> (REL VAR_1 ... VAR_n)
                  (instance-of VAR_m CLASS))))

> A kind of sort restriction? 

In the semantic sense above, yes; in a syntactic sense, no.

> If so, ...

I.e., if taken (incorrectly) in a syntactic sense, ...

> this doesnt do that job, since if one violates the sort restriction
> one simply generates a contradiction, 

As one should, if understood in a semantic sense.

> rather than a syntax error.

Correct.  The language in question, again, is not syntactically
sorted (better, I think, "typed").

> There is also a more subtle point, by the way, that this 'definition' 
> isn't a definition, 

It doesn't purport to be; as it says, it is a metalinguistic schema
that expresses in one fell swoop what an infinite number of
abbreviations.

> ...since it uses a metasyntactic convention (the 'triple dots')
> which isn't part of the language. 

Of course -- the same goes for the metalinguistic variables 'REL',
'M', 'CLASS', etc.

> One could define it in KIF 3.0 (but not in ANSI-KIF) by using
> sequence variables and recursion, but it still wouldn't be
> first-order definable.

Well, I'm not sure that's true, given a weak enough underlying
arithmetic (indeed, you could generate a weak theory in terms of
sequences, as we discussed at the TAMU KIF workshop); but that's
really a moot point.
