Just as a quick note to head off any possible misunderstandings, the 
term 'higher-order'is used in a different sense from this in logical 
circles (as in 'higher-order logic'). I mention this only because 
these two rather different senses of 'higher-order' are often 
confused. What you are calling 'higher-order' is usually referred to 
in logic using the prefix "meta-", as in 'metalanguage' = a language 
for talking about (another) language.  KIF 3.0 for example is most 
definitely not higher-order logic, but it does have enough 
meta-descriptive power to describe its own syntax (more or less; 
there are some technical snags...I'll get back to you on those 
later...)

'Reflection', by the way, is often used to refer to the ability of a 
language or system to not only describe its own signs, but to move 
between the signs themselves and their descriptions, so that one 
might for example be able to *infer* a description of a sign from the 
presence of the sign itself (that would be 'upward' reflection) and, 
more ominously for the possible inconsistency of the system, also 
have 'downward' reflection, ie be able to infer a sign from an 
assertion that the description of the sign was true.
