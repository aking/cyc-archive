I agree with Nicola's suggestion, which I also recommended
at various times:

>>I warmly suggested the same some time ago... More papers, less messages!

But I think that a different format would help very much.
Ideally, we should have papers submitted in a web-zine style,
such as Slashdot, for example.

The abstract for each paper would appear in the opening page
of the web-zine.  When you click on the URL, you get to the
paper itself, which is followed by a list of all the comments
on that paper.  At the end of the comment list is a window, in
which anyone can add further comments, which the software would
automatically add to the end of the comment list.

That is how the Slashdot software works, and it can be used
either for new articles submitted directly to the SUO server
or for relevant articles that appear elsewhere on the web.

John Sowa
