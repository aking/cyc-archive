Thanks Chris,
 
If you had a chance to read my recent message to Jon Awbrey [http://suo.ieee.org/ontology/msg02393.html] you will see that in the conflict between functions and the universal set, I would prefer to eliminate the universal set. After all, other than getting itself defined, it has not been involved in any axiomatic thing ;^) that I have written so far. In fact, I am not just interested in having power classes 2^A, I am also interested in having exponent classes B^A, function evaluation, and adjoints to binary functions (turning them into unary functions to exponents). These all seem to be quite useful as foundations. In short, I am interested in assuming that classes and their functions satisfy the axioms of a Cartesian-closed category. Do you see any problem with this?
 
One thing to note is the following. Although the following property instigated the problem with functions and has now assumed more hallowed status, it was not assumed as a guiding principle from the beginning.
 
CATEGORICAL PROPERTY: The (new-KIF) axiomatization for the ontologies in the lower-level structure (the lower metalevel) is strictly category-theoretic -- all axioms are expressed in terms of category-theoretic notions, such as the composition and identity of large/small functions or the pullback of diagrams of large/small functions; [no KIF] no axioms use explicit KIF connectives or quantification; and [no basic KIF ontology] no axioms use terms from the basic KIF ontology, other than pair bracketing '[-]' or pair projection '(- 1)', '(- 2)'. This would seem to extend to all ontologies for true categories (not quasi-categories) -- those categories whose object and morphism collections are classes (not conglomerates).
 
Originally all axioms for the ontologies in the lower metalevel (such as the ontologies for spans, hypergraphs and classifications) were written from scratch. Only after more foundational terminology was coined, was the code translated to a form that satisfied the categorical property. Although there are many examples (I particularly love the example of the Information Flow notion of classification extensionality which uses equalizers of class functions), a good standard example is set function composition.
 
Here are the original axioms for set function composition (translated to new-KIF and ignoring the laws of associativity and identities), clearly written from scratch. This uses KIF quantifiers and terms from the basic KIF ontology ('KIF$function' and 'KIF$signature').
__________
 
(KIF$function composition)
(KIF$signature composition function function function)
(forall (?f1 ?f2 (function ?f1) (function ?f2))
    (<=> (exists (?f (function ?f)) (= ?f (composition ?f1 ?f2)))
         (= (target ?f1) (source ?f2))))
(forall (?f1 ?f2 (function ?f1) (function ?f2))
    (=> (exists (?f  (function ?f)) (= ?f (composition ?f1 ?f2)))
        (and (= (source ?f) (source ?f1))
             (= (target ?f) (target ?f2)))))
(forall (?f1 ?f2 (function ?f1) (function ?f2))
    (=> (exists (?f (function ?f)) (= ?f (composition ?f1 ?f2)))
        (forall (?o ((source ?f1) ?o))
            (= (?f ?o) (?f2 (?f1 ?o))))))
__________
 
Here are the current (and hopefully final) axioms for set function composition, taken from the ontology for small sets and their functions. The 'SET' and 'SET.FTN' prefixes refer to the ontology for classes (large sets) and their functions, respectively. The last definitional axiom above is contained in the foundational definition of 'SET.LIM$pullback'. As you can see, this contains no KIF quantifiers or logical connectives and no terms from the basic KIF ontology. But it does contain new foundational terminology ('SET.FTN$opspan', 'SET.FTN$composition', etc.). So, these axioms satisfy the categorical property -- everything is strictly categorical.
__________
 
(SET.FTN$opspan composable-opspan)
(= (SET.FTN$opvertex composable-opspan) set$set)
(= (SET.FTN$opfirst composable-opspan) target)
(= (SET.FTN$opsecond composable-opspan) source)
 
(SET$class composable)
(= composable (SET.LIM$pullback composable-opspan))
 
(SET.FTN$function first)
(= (SET.FTN$source first) composable)
(= (SET.FTN$target first) function)
(= first (SET.LIM$pullback-projection1 composable-opspan))
 
(SET.FTN$function second)
(= (SET.FTN$source second) composable)
(= (SET.FTN$target second) function)
(= second (SET.LIM$pullback-projection2 composable-opspan))
 
(SET.FTN$function composition)
(= (SET.FTN$source composition) composable)
(= (SET.FTN$target composition) function)
(= (SET.FTN$composition composition source)
   (SET.FTN$composition first source))
(= (SET.FTN$composition composition target)
   (SET.FTN$composition second target))

> > In formulating axioms the Classification Ontology I have come across a
> > foundation question about the "power operator". I am following the
> > small-large approach to foundations as advocated by Adamek, Herrlich and
> > Strecker in section 0.2 of their book "Abstract and Concrete Categories"
> > 1990, ISBN: 0-471-60922-6, QA169.A3199 1989. The basic concepts are sets,
> > classes and conglomerates. One axiom is that sets are closed under power;
> > that is, if A is a set then P(A), the collection of all subsets of A, is
> > also a set. They seem to indicate that classes are not closed under power,
> > since they give P(U), the power of the universe, as an example of an
> > illegitimate conglomerate (legitimate conglomerates correspond to classes; a
> > conglomerate X is *legitimate* when there is a class Y and a surjection Y ->
> > X).
> >
> > So, is it ill-advised to assume that classes are closed under power?  What
> > precisely is the problem here?
>
> If by P(S), for a class S, you mean the class of all subSETs of S, then
> there is no problem with closing classes under power. In fact, in pure set
> theory, the set of all subSETs of U is just U, since every set is a set of
> sets and hence a subset of U.  But the assertion that P(U) is illegitimate
> suggests they are talking about defining P(S) for a class S to be the
> class of all subCLASSes of S, in which case one runs afoul of Cantor's
> theorem when S = U.   (By Cantor's theorem P(S) is larger than S, for any
> S.  (The proof would go through for classes no less than sets if P(S) is
> taken to be the class of all subclasses of S.)  So P(U) is larger than U.
> But, like any class, P(U) is a subset of U, and hence no larger than U,
> contradiction.)
