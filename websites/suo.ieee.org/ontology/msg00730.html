> Well, current KIF (ANSI-KIf or KIF 3.0)is an unsorted first-order
> language, so all quantifications are unsorted. It allows for what
> looks like sorting restrictions on quantifieres, but they are only
> 'macros' for the usual mapping of sort-restricted quantification into
> an unsorted logic, where one treats sorts as predicates and
> explicitly states the sortal restrictions as antecedents.
>
> Your style of axiom is the 'dual' of this, in a sense: it is intended
> to say that productRelation takes arguments of a certain sort and
> produces results of a certain sort. But what happens if one writes an
> assertion using 'productRelation' which violates these sorting
> constraints? In a genuine sorted logic, this would be syntactically
> illegal. In an unsorted logic (like KIF) it would simply be a
> contradiction, or have unintended consequences.

Yes. I needed to express some typing (sort restriction) here, and was just

But, since I want this to express semantic (and syntactic) legality, I agree
that there is some problem using this in unsorted KIF.

OK, good. Here is a slightly more complex example taken from an ontology for
category theory.

(KIF$Class Category)
(KIF$UnaryFunction source)

(forall (?c ?s)
    (=> (= (source ?c) ?s)
        (and (Category ?c) (KIF$UnaryFunction ?s))))

(forall (?c ?m ?o)
    (=> (= ((source ?c) ?m) ?o)
        (and ((morphism ?c) ?m) ((object ?c) ?o))))

Here is a version using the sorting predicate that you mentioned.

(SortOf source ((Category) KIF$UnaryFunction))
(forall (?c)
    (SortOf (source ?c)(((morphism ?c)) (object ?c)))

Here 'object and 'morphism' are some other KIF$Classes.
Does this look OK?

I am using the namespace notation 'namespace$name', since namespaces seem to
be very natural in this situation. The KIF namespace is for a core (or
bootstrap) KIF ontology. I have other namespaces for Categories, Functors,
etc. And for particular categories, such as Set and Classification. Are
namespaces to be included in SUO-KIF? I hope so!
