It seems to me that there are 4 notions here.

1. ontology naming

For example:

    (ontology A ...)

2. ontology locating with a location consisting of two parts
    i. a file pathname or URI path or something else
    ii. an ontology name
(here the file-name for the file holding an ontology is taken to be the same
as the ontology name)

For example:

    http://www.example.org/ontology/A

with pathname 'http://www.example.org/ontology/' and ontology name 'A'

3. ontology inclusion structuring

For example -- a module with two explicit submodules :

    (ontology B
        (ontology A ...)
        (ontology C ...)
     ...)

or a module with one implicit submodule specified using an inclusion
statement and one explicit submodule

    (ontology B
        (include A http://www.example.org/ontology/A)
        (ontology C  ...)
     ...)

4. namespace prefixing

For example:

    (namespace-prefix a A)

Then a relation named 'r' that is used in the A ontology can be referenced
locally in the module (ontology) that made the prefix declaration as 'a#r'.

__________

Perhaps then an import declaration such as

    (imports a http://www.example.org/ontology/A)

is a derivative notion that is conceptually equivalent to two statements, an
inclusion and a prefixing

    (include A http://www.example.org/ontology/A)
    (namespace-prefix a A)
