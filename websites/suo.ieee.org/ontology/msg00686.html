Matthew West says:

One of the things I have learnt is what I think you are suggesting here:
that if something is OK for a particular purpose, then it is OK for a
particular purpose, and this is OK.

The problem is that it might not be OK for another purpose, what then? 10
different models of the same thing each OK for its particular purpose? not
much of a standard I think. On the other hand One model into which each of
these 10 could be translated, allowing communication between them all could
possibly be very useful, and something worth standardising.
---

There seems to be an implicit assumption here that is it POSSIBLE to
have an ontology that is good for all purposes, or at least a very very
wide range of them.  I think there is an inherent generality/power tradeoff.
Only somethign very general can serve a very wide range of  purposes, but
then it will provide little traction. Somethgn very specific, designed for
a particular purpose will be more powerful, giving more traction, but will
serve fewer purposes. I see this a a fundamental tradeoff that will never
go away. Please correct me if this view is flawed.

Mike
