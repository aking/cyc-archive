Dear Chris,

Some more questions and comments.

1. Under class, relation is defined as a class. Later under relation you
define class as a unary relation. This seems to be inconsistent.

2. < applies only to integers. Why doesn't it apply to reals or (at least)
rationals?

3. < is not defined (not that I have any idea how to do it). By which I mean
what being less than is.

4. I can't get the axiom for addition (or subtraction). Here is my attempt
at an English translation - where have I gone wrong?

For all integers i and j
  i+0=0 and  <-- I think this should be i+0=i
  i+(j+1)= i+j+1 and
  i+(j-1)= i+j+1

Same applies to subtraction.

phew so all those years using reverse polish notation on HP programmable
calculators did have a point :-)

6. Sequences

a) Here you find a need to introduce extensionality for some classes like
sequence. Robert has suggested they should also be necessary for Top, or
else there is no reason for Top to be unique.

It is starting to look to me as if it would be better to introduce a
"property" like class that can be non-extensional, and a "set" like class
that is extensional before we get to this point.

b) on the one hand you say that a sequence holds only for positive integers,
and on the other hand you say that any object can be an element of a
sequence. Which is it?

c)
(forall (?s)
        (=> (sequence ?s)
            (and (relation ?s)
                 (binary ?s)
                 (functional ?s)
                 (exists (?n (posint n))  <-- missing ?
                         (and (not (exists (?x) (?s ?n ?x)))
                              (forall (?m (posint ?m) (< ?m ?n))
                                      (exists (?x)
                                              (?s ?m ?x))))))))

My translation is:

For all s:
  If s is a sequence then:
    s is a relation and
    s is binary and
    s is functional and
    for some positive integer n
      there is no x such that s(n,x) <-- doesn't seem to have
                                         a defined meaning
      and
      for all positive integers m where m<n
        there is some x such that s(m,x) <-- meaning?

c) you have used the term length to mean just the length of a sequence. This
means it cannot be used for e.g. length as a basic dimension. Aren't what
you are defining here as sequences really tuples? If not what is the
difference?
