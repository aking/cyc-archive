My brain is hurting but...

I am trying to figure out how the cactus language that you (Jon)
are proposing is going to provide that normal First Order Predicate Logic
doesn't.

If I want to say "all cats have a tail"
something like:

forall c 
   such that c is an instance of Cat
there exists a t
   such that t is an instance of Tail
There exists h
   such that h is a particular tuple/relational-instance of 'has'
There exists no more than one h
   whereby h expresses the connection between c and t

Now this is a rough way of expressing this.
How would the cactus language make this expression
   1) clearer
   2) more compact
   3) less ambiguous
   4) more powerful

or any other qualifier you would want to examine?

Right now, the cactus language looks like a lot of syntax to memorize
with no discernable advantage in expressing common facts about the world
to a computer.

Dave
