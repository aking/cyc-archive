Mike, et al.
But there seem to be some options about what the individual sibling
relations are in the real world (is this possible if there is a real world?)
As Fritz Lehmann has also noticed, there seem to be two camps using the same
word (tuples) to denote different things.
The logical view is that they are tuples - as Pat Hayes wrote:
Close. The relation is a *set* of triples: all the triples that the relation
holds true of. So for example if A sent Bill to B , then the triple <A,
Bill,B> is one of the triples in the "billing by post" relation-set. Of
course there are probably others in the same set.  Every time someone gets
billed, that defines another triple in the relation-set. Think of this set
of triples as God's database of all the billings that ever happened or ever
will happen. (It has to be God's since it might well be infinite.) As far as
logic is concerned, this (potentially infinite) set of triples *is* the
relation in question.
The relational database view is that they are a different kind of tuple -
where, somehow the tuple also carries content.
We can distinguish them by their identity conditions. If we swap the columns
in relational database we do not change the identity (or meaning) of the
tuple. If we swap positions in a tuple e.g. <A, Bill, B> to <B, A, Bill> we
do. Similarly while in logic a tuple can belong to a number of relation
classes, it is not clear that this is so for the relational view.
One advantage of the logic view is that relations can be treated like
properties - and there can be a simple taxonomy of relations. This is more
difficult to achieve under the relational view. However, the logic view does
lead to the seemingly odd conclusion that for every relation class and
particular relation there also exists its inverse.

Whatever you call it, there is something in the real world which entails two
things being related to each other in a particular way.  Take for exampl,
being ciblings.  I'm sorry, but I can see no way round ciblinghood being in
the real world.
MK> I absolutely agree
You can choose to call it a relation, or a relationship, or a frobnit, but
it is a real world notion, which from time to time one may wish to
represent.
MK> I choose to use the word relationship to refer to the thing in the real
world, and reserve it for only that. Conversely I choose the word relation
to refer to the construct in my model which represents the things in the
real world.  I sometimes want to talk about the one and sometimes the other
and find it essential to have different words to refer to them.
MK> A complication is that the word relation, in the singular, is used by
Ted Codd (and I believe generally by mathematicians) to mean the set of
tuples, whereas the word relationship, in the singular, in natural English
can only mean either a single instance, or a type of, relationship.  Thus a
relation (singular) actually represents many relationships (instances).
