Hello everybody,

We are designing and generating a set of testbeds to evaluate the
performance of our CGs processor. For that we would really appreciate some
actual data (or informed estimates) of various qualitative aspects and
quantitative parameters of relevant, industrial-size ontologies. Thus, if I
may ask, in the context of the ontology you have been developing or using:



What is the size of the type inheritance structure (lattice), i.e. how many
types (categories, classes, etc.) does the system have (or expect to have in
the medium/long run)?

Do you use (or expect to use) multiple type inheritance?

How much functionality/utility would you lose without multiple inheritance
for types?

What is the ratio of singly inerited types to multiply inherited types (if
used)?

Do you expect to use relations/predicates of arity higher than 3?
(Examples?)

What are the actual (or expected) ratios of relations of different arities?

Do you use (or expect to use) inheritance for relations?

Do you use (or expect to use) multiple inheritance for relations?
(Examples?)

How much functionality/utility would you lose by not using multiple relation
inheritance?

If so, what is the ratio of singly inherited relations to multiply inherited
relations?

How many type definitions?

What's the ratio of defined types to primitive (undefined) types?

How many relation definitions?

What's the ratio of defined relations to primitive (undefined) relations?

How many rules/constraints (max/average) associated with each type ?

How many rules/constraints (max/average) associated with each relation?

How many general rules (i.e. not type or relation - associated constraints)?

What is the max size of a rule antecedent (number of nodes)?

What is the max size of a rule consequent (number of nodes)?

Size of actual application fact base - how many different individual
concepts?

Size of actual application fact base - how many relations (i.e. all relation
occurrences)?

Maximum size of a query - how many concepts and relations in a query?

Maximum wait for a query to produce an answer?

Did I miss any important qualitative or quantitative aspects of your
ontology?



Thanks & best wishes,

Pavel
