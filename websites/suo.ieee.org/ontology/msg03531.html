JA = Jon Awbrey
SR = Seth Russell

JA: In this paradigmatic (analogical or exemplary) way of
    understanding it, a formal language is the "data" and
    a formal grammar is the "theory", and the question is,
    as always, whether a theory accounts for and explains
    the data, a "fitting" relationship that may be viewed
    in many ways, for one, the way that a theory might be
    said to "generate" the data, or perhaps better stated,
    not just to "cook" in a precociously specious fashion
    but more like to "regenerate" the form after the fact.

SR: The way I see this, (if indeed I am seeing the exact same techture),
    is to picture different Lego sets (different grammars) ... with some
    kinds of Lego sets you can build some kinds of things ... with others
    you build other kinds of things.  Are we looking at the same thing?

well, say we fix on a particular set !A! of alphabet blocks.
we want there to be this biggest possible, freely-generated
formal language X = !A!* that can be built from our building
blocks, and with lego blocks i would have to worry about all
of the extra built-in constraints from the get-go.

any old formal language L on this alphabet is L c X = !A!*.

now, there are always many different grammars G that would
generate L, but each grammar G generates only one language.

analogue would be:  many theories for each world of data,
but each theory accounts for precisely one world of data.

hmmm.  that sounds fishy, of course, but I think that the difference is this:
so far I am only admitting to my formal arena complete datasets (languages)
and complete theories (grammars).  it is possible to make this all more
realistic from an empirical standpoint if we really want to.

jon awbrey
