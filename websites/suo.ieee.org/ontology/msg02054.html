Dear Pat,

> >Dear Pat,
> >
> >I know you (I) need it, the question is whether I can 
> explain it adequately.
> > > >
> > > >The only thing not mentioned here that is important (in 
> my opinion)
> > > >is a usage tree for when objects that are "defined" in other
> > > statements
> > > >are used in this one. E.g.
> > > >(
> > > >REFNAME: Extensional-relation
> > > >USES: sequence-extensionality
> > > >...
> > > >)
> > >
> > > I can't quite see the point of this, since one can see what each
> > > axiom uses just by looking at the axiom.
> >
> >MW: What you can see in the axiom is the names of the other 
> constants that
> >are used in it, not where to find them. The vision I have is 
> that if you
> >have a
> >library of axioms in different stores (I leave open whether this is a
> >database or a file system) then what you have are the names 
> of the axioms,
> >not the names of the constants defined in those axioms. If you have
> >something like a USES: statement then when you identify that 
> for some piece
> >of reasoning you need some particular collection of axioms, you can
> >automatically pull in all the other ones that are relevant.
> 
> OK, I get the idea. I agree this would be useful, but it might be 
> something that goes beyond KIF itself. I see this as a tool added to 
> KIF. What we do need to do is to add 'hooks' to enable things like 
> this to be added to KIF painlessly.
> 
> Many existing ontology systems have 'knowledge browsing' facilities 
> like this. Cyc in particular has a very elaborate collection of tools 
> for finding all axioms used by a concept, all axioms connected to 
> another axiom, and so on.
> 
> >MW: The alternative as I see it is that for any piece of 
> reasoning you have
> >to pull everything in. I expect everything to be quite 
> large, eventually.
> 
> One thing we do have is the ability to give a bunch of axioms a name, 
> and then include them in another module by using the name.
> 
MW: As long as that was recurcsive, i.e. the module referred to would
pull in the modules it in turn referred to, then this would I believe
meet the requirement. How does the syntax look?
