I do not have any vested interests in KIF, but I did want to comment on my
recent, rather intensive experience with the USE of the new KIF in defining
ontologies -- I do not believe that you can really know a language's
strengths and weaknesses, until you have intensively used it. That said, as
an onto-logical MACHINE language, I find the Lisp-y style of logical
expression of the new KIF very useful, adequately expressive and
conveniently terse. I do not expect it to be used as an "industrial
strength" ontological language. What I expect is indicated by my previously
mentioned programming language analogy (see
[http://suo.ieee.org/email/msg04370.html]):
    * The new-KIF is like an ontological machine language.
    * The basic KIF ontology is like an ontological assembly language.
    * The metalevel (structural component) of the SUO is like a high level
programming language such as Lisp, Java, ML, etc.
    * The content component of the SUO is like the various software
applications, such as word processors, browsers, spreadsheet software,
databases, etc.
