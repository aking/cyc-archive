Pat,

I think that we have argued this point to the point of
exhaustion.  The difference between us is similar to the
difference between programming languages that pass "objects"
as arguments of functions and those that pass "pointers" as
arguments of functions.  In the actual implementation, the
underlying data structures can be very similar, if not
identical.

Your sets contain actual objects, and my sets contain
surrogates, which point to objects.  It makes a difference
in syntax, but the formal structures are isomorphic.

>John and I argued about this for years, but the answer seems to me to 
>be quite clear: yes, it can be a real world object. The Tarskian 
>model theory requires that the universe of discourse be a nonempty 
>set. That is (quite literally) ALL it requires: any set will do. In 
>particular, it can be a set *of anything*. It might contain cows or 
>molecules or gas clouds or human histories or electrons or flowers or 
>notions of beauty or anything you want it to contain. Of course, it 
>could also contain 'surrogates' such as symbols or integers, as well, 
>and there is a kind of sense in which John's substantive point is 
>well taken: there is nothing in the model theory which can possibly 
>*guarantee* that the universe of an interpretation shall contain 
>(say) cows rather than cow-surrogates. But there is nothing that 
>*forbids* it, either.
>
>Now, this doesnt make it impossible to discuss what John calls the 
>'real issues'; but I concede, it doesnt go very far towards solving 
>those issues either. It is simply agnostic about them.

John
