All,

           Could I suggest we take a break from talking about  the lists
and just give them a try.  We can always change them if needed.  But
first, here are some points.

1. We need to keep reminding ourselves that we are here to develop a
standard.  Postings to this main list should be related (perhaps directly
related) to making progress on the standard.

2. Some substantial content has been posted, yet is getting very little
review.  When I originally posted the Scope and Purpose I got no
response, but when we asked for votes with comments, we received many
good comments.  Does anyone have any suggestions on how we can generate
serious review and make progress on the posted content?

3. There are an unlimited number of topics that 'could' play into SUO,
but do these belong on this list?  I suggest they do if, and only if, a
direct connection to SUO is described.  For example, you could describe
how it impacts some of the content that has been proposed.  

4. I believe administrative issues do belong on this main list.  We need
a single list for the work of SUO, which includes both technical and
administrative issues.  

5. When replying to a message, please consider whether it needs to go to
160 people, or whether you can send it just to the sender.  

6. Please do not post the same message to more than one list.

Now, let's get back to business.

Jim Schoening
