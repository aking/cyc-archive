My idea is that the "real issue here" is not KIF but the structural ontology
[see my comments at the bottom
http://grouper.ieee.org/groups/ltsc/logs/ontology/msg00522.html of my
response to Pat]. KIF only needs to give us sufficient mechanisms that we
can represent enough foundational set theory in a core (or bootstrap)
ontology. This set theory will then allow us to define a suitable collection
of mathematical structural ontologies that enable "mechanisms for
partitioning an ontology or of noting dependencies between" ontologies. In
the message threads
[http://grouper.ieee.org/groups/ltsc/logs/ontology/msg00469.html] and
[http://grouper.ieee.org/groups/ltsc/logs/ontology/msg00471.html] a have
started to present such mathematical structural ontologies: Classification
Ontology (done in KIF), Core KIF Ontology (clearly necessary, but unclear on
actual content), Ontology for Categories, Functors and Natural
Transformations (done in KIF, not sent yet). There are at least 4 or 5
others that are necessary: Hypergraph Ontology (done on paper, not yet in
KIF), Language Ontology (a little delicate; done on paper, not yet in KIF),
Model Ontology (done on paper, not yet in KIF), etc. These have sufficient
"morphism and colimit structure" [see the message thread
http://grouper.ieee.org/groups/ltsc/logs/ontology/msg00375.html] to build
things.
