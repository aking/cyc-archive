Dear Chris,

I'm very pleased to see this.

> I'm inclined
> (following Pat Hayes) toward developing some sort of "wrapper" for
> ontology axioms (definitions, theorems) that encloses both 
> the axiom and
> its documentation and which avoids numerical references (or at least,
> avoids using them in any way that implies any sequential 
> ordering of the
> elements of the ontology), e.g.:
> 
> {
> REFNAME: sequence-extensionality
> STATEMENT-TYPE: axiom
> DOCUMENTATION: "Sequences are extensional, i.e., sequences 
> with the same elements in the same order are identical."
> CONTENT:
> 
> (forall (?seq1 ?seq2 (sequence ?seq1) (sequence ?seq2))
>         (=> (forall (?n ?x)
>                     (<=> (?seq1 ?n ?x) (?seq2 ?n ?x)))
>             (= ?seq1 ?seq2)))
> }

The only thing not mentioned here that is important (in my opinion)
is a usage tree for when objects that are "defined" in other statements
are used in this one. E.g.
(
REFNAME: Extensional-relation
USES: sequence-extensionality
...
)

Regards
