Josiane,
   I'd be glad to help you.  For starters, "on" could be converted to the 
SUMO predicate "on" (see 
http://ontology.teknowledge.com:8080/rsigma/SKB.jsp?req=SC&skb=Merge&id=648 )

"then" and "so" are a bit tricker since their translation to logic requires 
considerable context in their interpretation.  Often (although not always) 
they involve a statement of temporal precedence so you'd need to reify 
events and then relate those events temporally with the temporal bounds and 
relations like "before" (see 
http://ontology.teknowledge.com:8080/rsigma/SKB.jsp?req=SC&name=before&skb=Merge 
)

"take" and "put" also can be translated in a way similar to Davidson's 
event semantics (which I assume you're already familiar with - if not I can 
dig up some pointers).  You'd need to relate events with the agents and 
objects that participate in those events.  Start by looking at CaseRole 
(see 
http://ontology.teknowledge.com:8080/rsigma/SKB.jsp?req=SC&skb=Merge&id=213 
).  This is an area that needs some fleshing out in SUMO so it would be a 
great area for you to focus on.

My belief is that although the labels on terms in SUMO are English-based 
and therefore much easier for English-speaking ontologists, that we're 
trying for an ontology that does not inherently encode an orientation that 
is particular to any one human language.  Your input will help ensure that 
we work towards that goal.  In fact, it would ultimately be a very useful 
exercise to have equivalent versions of the SUMO in several different 
languages, where all the terms and axioms would be identical except for the 
labels given to those terms.

Adam
