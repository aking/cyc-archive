Dear Jon,

I continue the struggle through the mists of ambiguity - 
hopefully towards you rather than away.

I think I am going to start again, trying to adopt some of 
the things you have suggested.

Purpose:

To take what is represented by an n-ary relation, and 
represent it by n binary relations and a set/class of objects.

Method:

For each tuple 

<component1, component2, ...>

that is a member of the n-ary relation (R) create exactly
one object (x) that is a member of the set of objects (S).

For each component of the tuple create a binary tuple that
is a member of a relation that indicates the place in the original
tuple that the component stands in. e.g.

place1 <x, component1>

e.g. if I have:

 (gives1 John Mary Book1)

If I apply the above method I might get:

(gives2 x)
(giver-in John x)
(receiver-in Mary x)
(given-in Book1 x)
