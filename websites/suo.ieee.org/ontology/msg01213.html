The basics of membership _do_ apply.

However, there are many set-theoretic _kinds_ of things.

In this approach, as you say, classes are not "things"; only ur-elements and
sets are things. Classes are subcollections of things.

This *Thing* collection kind is useful for the object language level, the
level embodied by the classifications in the Classification category, where
only atomic things and sets are relevant (think of a classification as
comparable to an ontology) -- the collection of types (which might include
ontological classes) is a set, the collection of instances is a set, and the
instantiation-predication relation is a binary relation between sets. I
guess this would correspond to the common working world, an e-commerce
ontology or a bioinformatics ontology or a particle physics ontology, where
set-theoretic classes, such as the set of all sets, are not usually (never?)
found.

However, if we found it useful, we could define a larger kind of thing
represented by a *THING* collection kind that contains as members of either
ur-element or classes. Perhaps this would be called "the next big(ger)
thing" ;^) But, I do not know how useful this would be for the common
working person; someone like an Irish carpenter (to use a Pat Hayes
example).
