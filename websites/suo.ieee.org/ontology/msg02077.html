MW: Why do you think anyone would want to say this? If you are going
to take a simple KIF binary relation instance and represent it in a
relational
database, and you want to do some basic data management of that
relation, you need several columns to do it. You need 3 columns (plus the
table name) as a minimum to have the equivalent expressive power in
a single record to something like:

(R x y)

Now in KIF when elsewhere you want to refer to (R x y) you just state it 
again there. In a database you don't, you make a surrogate for the object 
that is represented, and use that. So if KIF used a similar approach you
would have something like:

#1 (R x y)

Well you would need to be pretty dumb not to see that this is 4 elements 
you need to find a place for. Even with one of them being the table name
you are left with 3 (minimum).

So to suggest that there is an equivalence between a binary relation and 
a table with 2 columns is either ignorant or mischevous.
