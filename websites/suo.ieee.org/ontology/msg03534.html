What is the utility of defining mentographic arrows in your Cactus
formalism?  What will this buy us?  Doing that seems to me to be like
defining Fortran in Cobol.   What I need is a critique of the definition I
gave and some constructive suggestions as to what else is needed.

> if you have a string form, then it usually pays
> to define both the parse/pointer structures and
> the char string versions in parallel.

Well there is a super simple string version, call it Pentuples.  Refer back
to the original post for the definition and restraints on the terms C, S, V,
and Q:

Sentence ::= C ws S ws V ws O ws Q  {ws | null}  '.'
ws  is any number of spaces or tabs or lfs.
Mentograph ::= Sentence*

This Pentuples language is similar in grammar and purpose to the NTriples
language that is used for the RDF Model Theory.
http://www.w3.org/2001/sw/RDFCore/ntriples/

Of course Pentuples is just a way to encode a mentograph in a string, it is
not intended to be a practical language in which to communicate.   To
communicate mentographs in strings between systems you can use any language
whatsoever for which you have a parser and which is sufficiently expressive
for the particular graph   ... RDF, KIF, N3 or Semenglish, or English are
all good choices.

For Semenglish I've given a formal grammar in a mentograph ... please see:

http://robustai.net/mentography/SyntaxSemenglish7.gif
and the BacusNaur structure used by that graph
http://robustai.net/mentography/SyntaxBacusNaur1.gif
also a graph showing the correspondence between Semenglish and the graphical
pictures I draw.
http://robustai.net/mentography/MentographySemenglish.gif

An constructive criticism at all ?
