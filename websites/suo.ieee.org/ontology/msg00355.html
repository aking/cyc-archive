>Pat,
>
>I think that we have argued this point to the point of
>exhaustion.

Indeed.

>The difference between us is similar to the
>difference between programming languages that pass "objects"
>as arguments of functions and those that pass "pointers" as
>arguments of functions.  In the actual implementation, the
>underlying data structures can be very similar, if not
>identical.

I agree, this isnt a debate about datastructures. Herbrand's theorem 
can be read as showing that any first-order semantics can be 
'imitated' by one constructed entirely from textual surrogates, in 
the sense that a sentence is satisfiable in one sense iff it is 
satisfiable in the other.

>Your sets contain actual objects, and my sets contain
>surrogates, which point to objects.  It makes a difference
>in syntax, but the formal structures are isomorphic.

Well, yes and no. First, I think it is important when discussing 
ontology to bear in mind that sets are usually thought of as not 
imposing any type constraints on their members; so anyone who imposes 
such a constraint needs to be very explicit about it, so that others 
can translate that notion of set into the more general notion, just 
to keep the terminology straight. Second, it can make a genuine 
difference when we are discussing semantic issues, or meta-logical 
issues more generally. For example, if the set of surrogates is 
countable (as they usually seem to be, eg sets of integers or sets of 
expressions) then one does get a slightly different model theory, and 
issues of completeness and decideability get altered. And third, on a 
philosophical level, I think that the more restricted view of sets 
forces one to adopt conclusions about the restricted application of 
some theories of truth which are not really warranted, such as that 
model theory cannot deal with references to real things.

But let us agree to disagree. I don't want to re-open this old 
debate, only get my cards onto the table.

Pat
