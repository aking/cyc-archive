> 
> > > Saying what a relation is does not automatically say that 
> all things
> > > like that must be relations.  If every set of tuples was 
> a relation,
> > > then we would have a genuine higher-order logic;
> >
> >MW: Excuse my ignorance, but why does this make for a higher 
> order logic?
> 
> Because it would imply (if taken seriously as a semantic constraint) 
> that the universe of quantification for relation variables must be 
> the actual power set of *all* sets of  n-tuples of elements in the 
> base universe, and with that semantics we would have genuine 
> (non-semi-decideable) higher-order logic. See 
> http://suo.ieee.org/email/msg00345.html for an intuitive discussion.
> 
> > > but we do not make
> > > this assumption (though it is, I concede, often made by
> > > mathematicians.) So it is quite possible for a relation 
> to exist but
> > > its inverse not to, or at any rate not to be considered a 
> relation in
> > > the universe of all relations.
> >
> >MW: And presumably this is sufficient to make the logic 
> first order. Could
> >you explain this in terms I might understand?
> 
> What makes it first order is that it makes no particular assumptions 
> one way or the other about the set of 'all relations'. The universe 
> of relations might be every set of n-tuples, or it might be so small 
> that it only contains a referent for the particular relation symbols 
> in the language. Unlike higher-order logic - which assumes that 
> *every* set of n-tuples is a genuine relation, so when you write 
> (forall (?r).... ) you are making a very large claim indeed - the 
> first-order version uses the same syntax, but understands it much 
> more modestly. It only requires that the universe of relations 
> contains at least enough things to be the referents of the names in 
> the language.

MW: handily that is probably sufficient.
> 
> Note however that all through this discussion, 'relation' means set 
> of n-tuples. This discussion is only about how many of those sets we 
> want to count as 'genuine' relations, ie how many of them our logic 
> claims to range over when it says 'forall ?r...'.

MW: This sounds a bit like the way I talk about classes and sets, where it
boils down to the classes are the sets we are interested in. In this case
there seems to be additional significance around what it takes to get to
higher order logic.
