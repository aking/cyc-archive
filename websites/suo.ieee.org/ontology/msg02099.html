Pat,

One other point.

Considering some of the examples in the ontologies on which I am working, it
would be quite useful to be able to use a relative namespace prefix with the
terms in the included subontology.

For example, consider the following situation.
    * the ontology B is located at B-place
    * a relation named 'r' is used in the B ontology
    * the ontology C is a subontology of ontology B with a relative
namespace prefix 'c1' and relative locator C
        (ontology B ...(ontology c1 C ...)...)
    * a second distinct relation named 'r' is used in the C ontology.

In the local context of B
    * in B the relation named 'r' in B is referred to by the name 'r'
    * in B the relation named 'r' in C is referred to by the name 'c1#r'

In the local context of C
    * in C the relation named 'r' in C is referred to by the name 'r'
    * in C the relation named 'r' in B is either not referencable or is
referred to by some name such as '#r'

In the global context consider first importing all of B
    * the ontology A contains an importing expression '(imports b B-place)'
Then
    * in A the relation named 'r' in B is referred to by the name 'b#r'
whereas
    * in A the relation named 'r' in C is referred to by the name 'b#c1#r'

In the global context next consider importing only the subontology C
    * the ontology A contains an importing expression '(imports c2
B-place#C)'
Then
    * in A the relation named 'r' in C is referred to by the name 'c2#r'

As an actual example, in my Set Ontology I define a 'product' term for sets
and I define a 'product' term for functions. Also, in the Set Ontology I
define a 'vertex' term for limits and a 'vertex' term for colimits. My
thought was to make subontologies for functions, limits and colimits.
