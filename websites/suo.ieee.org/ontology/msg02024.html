Here is a rough sketch of a proof of
the "Case Analysis Principle" (CAP),
the "Case Analysis Theorem"   (CAT),
as the case may be, however you might
find it most convenient to think of it.

In order to think of tackling a proof of this result,
we need to add a number of axioms and axiom schemata.
Because I abandoned proof-theoretic purity somewhere
in the midst of putting this system on the computer,
I never got around to finding the most elegant and
minimal, or even a complete set of axioms for the
so-called "Cactus Language" that RefLog employs,
so what I list here are just the rudiments of
the hodge-podge that I have found over time
to be necessary and useful in most settings.
Some of these special "axioms" are probably
provable from genuine axioms but I have yet
to go looking for a more proper formulation.

---------------------------------------------------
 Axiom L1, Indifference.  Merge -> / <- Split
---------------------------------------------------
|
|           x
|           o
|       x   |
|       o---o
|        \ /
|         @              =              @
|
|      (x, (x))          =           [blank]
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

---------------------------------------------------
 Axiom L2, Equality.  The Following Are Equivalent:
---------------------------------------------------
|
|           y          x   y          x
|           o          o---o          o
|       x   |           \ /           |   y
|       o---o            o            o---o
|        \ /             |             \ /
|         @       =      @      =       @
|
|      (x, (y))   =  ((x , y))  =   ((x), y)
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

---------------------------------------------------
 Axiom L3, Dispersion.  Disperse -> / <- Distill
---------------------------------------------------
|
| For k > 1, the following equation holds:
|
|   y1    y2  ...   yk       x.y1  x.y2  ... x.yk
|   o-----o...o-----o         o-----o...o-----o
|    \             /           \             /
|     \           /             \           /
|      \         /               \         /
|       \       /                 \       /
|        \     /                   \     /
|         \   /                     \   /
|          \ /                       \ /
|         x @            =            @
|
|  x (y1, ..., yk)       =    (x.y1, ..., x.yk)
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

To see this, if x is true, then it makes no difference
on either side of the equation, but if x is false, then
both sides are false.  There can be only one, remember.

NB.  The argument that I just gave for this form of distributive law
depended on the truth-functional interpretation of the lobe operator,
so this marks a patch that will need to be fixed up at a later point.

Anyway, these three axioms in addition to the rest,
ought to be enough to find the proof of this truth:

------------------------------------------------------
 Case Analysis-Synthesis.  Analyze -> / <- Synthesize
------------------------------------------------------
|
|                                         x
|                                         o
|                                     x   |
|                              Q[o/x] o---o Q[|/x]
|         Q                            \ /
|         @              =              @
|
---------------------------------------------------
|
|         Q              =  (x Q[o/x], (x) Q[|/x]) 
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

Proof Sketch:

¤~~~~~~~~~¤~~~~~~~~~¤~~~DEQ~~~¤~~~~~~~~~¤~~~~~~~~~¤
|
|                        Q
|                        @
|
=================================================== L1 [Split @]
|
|                          x
|                          o
|                      x   |
|                      o---o
|                       \ /
|                      Q @
|
=================================================== L3 [Disperse Q]
|
|                          x
|                          o
|                      x   |
|                    Q o---o Q
|                       \ /
|                        @
|
=================================================== C1 [Reflect x]
|
|                          x
|                          o
|                      x   |
|                    Q o---o Q[((x))/x]
|                       \ /
|                        @
|
=================================================== C2 [Weed x, (x)]
|
|                          x
|                          o
|                      x   |
|               Q[o/x] o---o Q[|/x]
|                       \ /
|                        @
|

The preparatory materials that we need are these:

[ Note On Usage:
|
| I am going to revert to my customarily sloppy workshop manners
| and refer to propositions and proposition expressions on rough
| analogy with functions and function expressions, which implies
| that a proposition will be regarded as the chief formal object
| of discussion, enjoying many proposition expressions, formulas,
| or sentences that express it, but worst of all I will probably
| just go ahead and use any and all of these terms as loosely as
| I see fit, taking a bit of extra care only when I see the need.
|
| Pet Names:
|
| Strings and graphs sans labels are called "bare".
| A bare terminal node, "o", is known as a "stone".
| A bare terminal edge, "|", is known as a "stick".
]

---------------------------------------------------
 Evaluation Rule
---------------------------------------------------
|
| The formal rule of evaluation for a k-operator is:
|
| ( x1, x2, ..., xk )  =  [blank]
|
| iff
|
| Just one of the arguments x1, x2, ..., xk  =  ()
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

---------------------------------------------------
 Interpretation Rule
---------------------------------------------------
|
| A "lobe operator" of the form "(x<1>, ..., x<k>)"
| enjoys two commonly employed interpretations for
| propositional logic, that is, ways of reading it
| as an assertion about, or a constraint upon, the
| logical values of the ordered arguments, that is,
| the mentioned variables x<j> for j = 1 through k.
|
| Existential Interpretation:
|
|     "Just one of the k arguments is false."
|
| Entitative  Interpretation:
|
|     "Not just one of the k arguments is true."
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

Let Q be a proposition with an unspecified, but context-suitable
number of variables, say, none, or x, or x<1>, ..., x<k>, as the
case may be.  (See what I mean?  I should've said "sentence Q".)

Let the "replacement expression" of the form "Q[o/x]" denote
the proposition that results from Q by replacing every token
of the variable x with a blank, which is to say, erasing "x".

Let the "replacement expression" of the form "Q[|/x]" denote
the proposition that results from Q by replacing every token
of the variable x with a stick stemming from the site of "x".

In the case of a proposition Q, that is to say,
an expression of it, not having a token of the
specified variable x, let it be convened that
that Q[o/x] = Q = Q[|/x].

---------------------------------------------------
 Axiom I1.  Condense -> / <- Distract
---------------------------------------------------
|
|     o   o         o
|      \ /          |
|       @     =     @
|
|    ( ) ( )  =    ( )
|
---------------------------------------------------
 Axiom I2.  Refold -> / <- Unfold
---------------------------------------------------
|
|       o
|       |
|       o
|       |
|       @     =     @
|
|     (( ))   =
|
---------------------------------------------------
 Axiom J1.  Delete -> / <- Insert
---------------------------------------------------
|
|      xo           o
|       |           |
|      x@     =     @
|
|     x(x)    =    ( )
|
---------------------------------------------------
 Axiom J2.  Distribute -> / <- Collect
---------------------------------------------------
|
|     y   z      xy   xz
|     o   o       o   o
|      \ /         \ /
|       o           o
|       |           |
|      x@     =     @
|
|   x((y)(z)) = ((xy)(xz))
|
---------------------------------------------------

The first theorem is known as the "Double Negation Theorem" (DNT).
The proof that succeeds it is derived from the one that was given
by George Spencer Brown in his book 'Laws of Form', and credited
to two of his students, John Dawes and D.A. Utting.  This result
is annotated as "Consequence 1" (C1) or as "Reflection" in LOF.

------------------------------------------------------
 Double Negation Theorem C1.  Reflect -> / <- Reflect
------------------------------------------------------
|
|       x
|       o
|       |
|       o
|       |           x
|       @     =     @
|
|     ((x))   =     x
|

One theorem of frequent use is what I like to call
the "Weed and Seed" Theorem.  The proof is just an
inductive exercise, so I will let it go till later,
but it says that a label can be freely distributed
or freely erased (retracted or withdrawn) anywhere
in a subtree whose root is labeled with that label.
The second theorem on the list to be shown here is
actually the base case of this Weed & Seed theorem.
I talk about it under the LOF name of "Generation".

-------------------------------------------------------
 Generation Theorem C2.  Regenerate -> / <- Degenerate
-------------------------------------------------------
|
|      yo         xyo
|       |           |
|      x@     =    x@
|
|     x(y)    =   x(xy)
|

|

Just to gather everything on one page, here is
the proof of the Splendid Theorem from Leibniz.

---------------------------------------------------
 Theorema Praeclarum (Leibniz).
---------------------------------------------------
|
|     r o   o s     o rs
|       |   |       |
|     p o   o q     o pq
|        \ /        |
|         o---------o
|         |
|         |
|         @                   =         @
|
|   ((p(r))(q(s))((pq(rs))))  =      [blank]
|
¤~~~~~~~~~¤~~~~~~~~~¤~~~DEQ~~~¤~~~~~~~~~¤~~~~~~~~~¤
|
|     r o   o s     o rs
|       |   |       |
|     p o   o q     o pq
|        \ /        |
|         o---------o
|         |
|         |
|         @
|
=================================================== C1 [Reflect "pq(rs)"]
|
|     r o   o s
|       |   |
|     p o   o q
|        \ /
|      pq o---------o rs
|         |
|         |
|         @
|
=================================================== [Weed "p", "q"]
|
|     r o   o s
|       |   |
|       o   o
|        \ /
|      pq o---------o rs
|         |
|         |
|         @
|
=================================================== C1 [Reflect "r", "s"]
|
|    pqrs o---------o rs
|         |
|         |
|         @
|
=================================================== [Weed "rs"]
|
|    pqrs o---------o
|         |
|         |
|         @
|
=================================================== C3 [Recess "pqrs"]
|
|         o---------o
|         |
|         |
|         @
|
=================================================== I2 [Refold "(())"]
|
|         @
|
