Let us now interrogate the alleged theorem that was lined up last time.
I will proceed quasi modo the "Case Analysis-Synthesis Theorem" (CAST),
that is to say, start out in a way that is model-wise if proof-foolish,
just because this is the manner that requires the least foresight, and
I am not feeling very promethean today.  One lemma that I can see just
far ahead enough to see that with which we need provision ourselves is
this principle here, that a bare lobe expression like "( , ... )" with
any number places for arguments but nothing but blanks to fill them up
is logically tantamount to the proteotypical expression of its species,
to wit, the constant expression "()" that an existentialist interprets
as denoting the logical value "false".  To depict it in graphical form:

¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

          o       o---o     o-o-o
          |        \ /       \ /
          @    =    @    =    @    =   ...

¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

Let us for the nonce dub this the "emptiness rule"
and worry later how to enscounce it in our system.
Still, another rule that we need is the following:

¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

                  x   x     x x x
          o       o---o     o-o-o
          |        \ /       \ /
          @    =    @    =    @    =   ...

¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤~~~~~~~~~¤

This is easy enough to derive from rules that are
already known, but call it the "indifference rule"
just in pursuit of ready reference and employment.

Well, I am departing from my computational intent,
so let me introduce a rule that is more suited to
that purpose, and which replaces the one just met
in many of the cases where we actually call on it:

---------------------------------------------------
 Evaluation Rule.  Spike -> / <- Setup
---------------------------------------------------
|
|   o
|   |   x2      xk
|   o---o...o---o
|    \         /
|     \       /
|      \     /
|       \   /
|        \ /                        x2 ... xk
|         @              =              @
|
| ((), x2, ..., yk)      =          x2 ... xk
|
---------------------------------------------------

I think this will be enough of a grubstake for us
prospectors of truth to move out on our adventure.
