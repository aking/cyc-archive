Adam,

>I'm all for good technology solutions but isn't the easiest solution
> just to have folks use the other lists that we've set up?

I realize that setting up a newslist is more work that an email list,
but it's the only way to handle a large number of participants.  If
the SUO had a good newslist, we could easily have a much larger number
of regular visitors.  And if the SUO is going to succeed, the number
of participants should really number in the thousands.  (Potentially,
it's a much bigger topic than Java -- if it were really done right.)

> There's been a
> rather large number of postings recently on Piercian scholarship that,
> while interesting, properly belong on ontology@ieee.org.  If that topic
> alone were moved over to the other list, we'd see a substantial shift in
> volume.  Since you are an acknowledged authority on that topic, I suspect
> that if you lead by example, others will follow.

The Peirce topic is just one example (and I've been overwhelmed
by the volume myself).  I would much rather read through that topic
in a Slashdot-style format than in the present email format.

It really doesn't matter which topic gets hot.  Any hot topic is going
to generate far more email than anyone can tolerate in e-mail format.
