Adam,

I agree:

>It looks like we're in agreement on the ultimate point, that we need a
> context mechanism.  As to how many alternate theories we need, we have a
> different intuition but it looks like we agree that it will be an empirical
> issue rather than one we have to decide up front.

As a general principle, whenever you are trying to decide
that you need X of some items for any computer related
application, any value for X that is less than infinity is
probably going to be inadequate.

That doesn't mean that we have to develop an infinite number
of microtheories, but that we should not assume that there is
any a priori limit on how many we will eventually need.
