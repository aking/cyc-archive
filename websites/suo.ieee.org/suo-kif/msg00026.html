Marta Giorgetti's point is important:

>I think that a CG considered as a literal is equivalent in KIF language
>to a proposition preceded by the quote operator, while  a CG considered
>as a descriptor is equivalent in McCarthy context language to a
>proposition that is true in some context.

That is exactly what I intended to say in Ch 5 of the KR book.

>I want to stress, as I showed above, that there is a difference in
>meaning between a literal CG and a descriptor CG.  So I wonder if the
>two kinds of CG's should be syntactically distinguished.

Yes, some kind of syntactic distinction is essential.  In a
programming language, there are several ways of making such
distinctions.  One way is to use an explicit operator, such
as a quote.

But in a language with strict typing, it is possible to take
advantage of the types to determine how a particular piece of
data should be interpreted.  If the data is not in the format
expected for the type, the system can automatically convert
it to the proper form.  Such a conversion is usually called
"coercion" in programming languages.

In conceptual graphs, several kinds of coercions have been
used since the very beginning.  A name, for example, is not
a unique identifier (as people named John are well aware).
Therefore, the concept [Person: John] is an abbreviation
for a graph of the following form:

   [Person]->(Name)->[Word: "John"],

which may be read:  There exists a person whose name is the
word "John".

When a CG occurs in the referent field of a concept, it can
only be interpreted as a CG if the type label is CG or some
supertype, such as Graph.  In all other cases, it must be
coerced to some other type.  As an example, the following
concept states that there is a situation of a cat on a mat:

   [Situation: [Cat]->(On)->[Mat]].

Since a graph is not a situation, this concept can be expanded
(i.e., coerced) into the following form:

   [Situation]->(Dscr)->[Proposition: [Cat]->(On)->[Mat]].

This says that there exists a situation described by (Dscr)
a proposition that a cat is on a mat.  But as Marta observed,
a graph is not a proposition.  Therefore, another coercion
is necessary:

   [Situation]->(Dscr)->[Proposition]->(Stmt)->[CG:
      [Cat]->(On)->[Mat] ].

This says that there exists a situation described by a
proposition, which has a statement (Stmt) by the CG
for a cat on a mat.

In the next version of the CG standard (which I am now writing),
I plan to explain these coercions more clearly.
