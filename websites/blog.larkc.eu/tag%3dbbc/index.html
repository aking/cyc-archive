Museums using non-curated Web-data

Sunday, March 22nd, 2009

Image via Wikipedia

(by Frank van Harmelen)

In Europe (not sure how the situation is elsewhere), the Cultural Heritage sector is very active and interested in using Semantic technologies, for linking across collections (e.g. the Europeana ThoughtLab), for personalised search, and more recently also for enriching their own curated digital collections with non-curated data from the Web. In an earlier post, we already pointed to what the BBC is doing on their Artists pages. Now, the Tate has started to enrich is digital collection of Turner’s landscape paintings with links from Google Street View, so that you can compare what Turner painted with how the same place looks today.

See a write-up in the Guardian, or try for yourself. This is all handcrafted, not using open Web of Data standards etc, so there is still a lot of work to do, but it’s a great motivating use-case. And why stop at Turner?

PS: since recently, the more-than-excellent Zemanta blog plugin that we’re using to enrich this blog also cleverly maps to Google Map entries: see the mention of the Tate above, Zemanta recognised the name and suggested the link (just as it suggested to use the above picture, by the way).

PPS: In an entirely non-scientific experiment I pitted Zemanta against Open Calais on a few general texts that I knew well, and I was much impressed by the breadth of coverage of Zemanta and the accuracy of its suggested links, tags and images.

BBC using Semantic Web technology in earnest

Monday, March 2nd, 2009

BBC (via last.fm)

(By Frank van Harmelen)

Two very good write-ups on how the BBC is now using Semantic Web technologies (RDF) and Linked Open Data (MusicMoz) in earnest:

A good general story, by BBC’s Tom Scott. One of the most convincing stories why a large organisation that doesn’t care about Semantic Web technology per-se is turning to adopt it, just because it makes its own mission easier and more effective.

Also, Marc Hibbins has on his blog a bit more techie explanation of how the Beeb’s Artist Pages (in beta since July’08) work. Not only is the BBC using public data, they’re also publishing it back on the web again (take the URL of any artists BBC page, put “.rdf” at the end, and check out what happens, check out e.g. this one). See the same blog-entry for good pointers to more material on this great Semantic Web use-case.
