Semantic Mashup of Biomedical Data

This will be of interest to LarKC-ers, given our use cases - a JBI special issue on the Semantic Mashup of Biomedical Data. There is a huge number of biomed data sources (Vassil gave a count of > 1000 at the Amsterdam meeting), and a constant need to integrate and work across several at once. For our GWA use case, for example, we will need to integrate and query over half a dozen data sources, together with text annotated against half a dozen others.

The special issue editorial talks about how the usual integration “by hand” (i.e. ad-hoc perl scripts) is now moving towards the use and integration of data sources with defined semantics, in rdf and owl, queried using sparql etc. The various articles give examples of such mashups - at first sight, using varying degrees of semantic technology.

Perhaps I’ll get round to reading some of the papers and posting a bit more…
