Easier plug-ins with Java 7

Monday, February 15th, 2010

(by Michael Witbrock, Cycorp Europe)

Image via Wikipedia

Image via Wikipedia

The LarKC architecture separates the platform, which provides services to support large scale inference, from the plug-ins, which implement novel methods of inference, data identification, data selection, transformation and work flow control. But maintaining this clean separation has been a little awkward, since one has to do a little bit of work to register new plug-ins with the platform or de-register them. This process looks set to get easier with Java 7, whose new watch service will make it easy to detect file system events, independent of OS.  Registering a plug-in should be as easy as dropping its JAR in a directory; De-Registering as easy as moving the JAR elsewhere.

alpha Urban LarKC online!

Tuesday, July 21st, 2009

(by Irene Celino)

After solving some problem on the demo server, finally the client application of the alpha Urban LarKC is available online!

  • The client application can be found here.
  • A demonstration video that explains how it works is available here.
  • More details on what’s behind the client application are given on the project wiki on this page.

Short intro: the alpha Urban LarKC client is a Urban Computing application to find monuments and events in Milano and to suggest the path to reach them. It is (obviously) based on the LarKC platform, which is able to answer SPARQL queries by sending requests to both Semantic Web sources (like Sindice and DBpedia) and Web 2.0 services (like Eventful), by processing data and finally giving back useful information to the client application which displays them on Milano map.

The good news is that the alpha Urban LarKC is likely the longest-running LarKC instantiation till now, continuously running since mid-June - unless you enthusiastically over-load it with your requests
The bad news is that the latest release 3.3 of DBpedia does not include anymore the geo-coordinates of a lot of Milano monuments which were available in the previous version, thus preventing the alpha Urban LarKC from displaying them…

Any feedback on the application will be more than welcome!

LinkedGeoData and Urban Computing in LarKC

Friday, July 10th, 2009

(by Irene Celino)

Yesterday, linkedgeodata.org announced their official launch.  They made available both a large dataset derived from the OpenStreetMap database and converted into RDF (which will contain around 2B statements, when the data conversion will be completed) and a set of REST/linked data services for online access to the dataset.

This is another sign of the growing interest for the location-based services on the Web and the data integration based on geo-spatial information.

What they did with OpenStreetMap is very similar to what we did in LarKC for the so-called “alpha Urban LarKC” - the first Urban Computing demonstrator build over the LarKC platform. We got  data from Milano municipality and converted them into RDF in order to integrate them with other data coming either from the Semantic Web (through Sindice and DBpedia) or from the Web 2.0 (via Eventful REST services).

The initiative of LinkedGeoData therefore represents a step further to a more comprehensive location-based Web and another opportunity for the Urban Computing in LarKC.

Qualitative Spatial and Temporal Reasoning

Thursday, March 26th, 2009

(Michael Witbrock, Cycorp) Many of our most basic thoughts involve temporal and spatial reasoning: putting something in the microwave; deciding to get out of bed  tomorrow morning; assembling examples in a list. One way of reasoning about these sorts of things is to place everything at an exact set of coordinates in time and space, but this doesn’t often lead to efficient ways of combining reasoning about time an space with reasoning about the things that are in time and space. Using qualitative spatial and temporal relations is helpful, and is generally what people do: we talk about things being next to one another, or after one another. We use deliberately vague expressions like “somewhere in Slovenia” or “somewhat later”.Giving computers the ability to reason with these qualitative spatiotemporal relations has been a research areas for almost as long as giving computers the ability to reason has. And a great deal of progress has been made.I recently gave a talk on integrating this reasoning with more general, broad covering reasoning systems (like Cyc), and encouraged participants at the Symposium to wrap their algorithms as LarKC plugins.  This is exactly the sort of community LarKC is intended to address:  researchers who would like to apply their results more broadly, and evaluate them in realistic, integrated settings, but who don’t have the resources (or, really, the motivation) to spend the time to build a platform. Judging from the enthusiastic response, I think we’ll see some spatial and temporal reasoners as early plugins.

Semantic Web for Transparant Government (again)

Saturday, March 21st, 2009

(by Frank van Harmelen)

Following our earlier posts on the use of Semantic Web technologies to make Governments more transparent (in July and February), there was much ado in the past couple of days on how the new US Government will use Semantic Web technologies for transparency about its economic recovery programme (e.g. on SemanticWeb.com and Sitepoint.com). However, currently it’s more promised than delivered. In the FAQ on Recovery.gov, we find:

Q: Is the spending data on recovery.gov available in a format (like XML) that developers can use to create mashups and gadgets?
A: Not at this time. But, as new systems are developed to capture the allocations and expenditures under the Act, we plan to make that data available in exportable form.

Hmm… Although the first US. Chief Information Officer (and responsible for setting up Data.gov) is talking all about XBRL formats and conversions to URI’s and RDF. I’d be interested to hear if anybody knows the timeschedules for this?

Open Calais no longer alone

Friday, March 6th, 2009

Image via CrunchBase

(by Frank van Harmelen)

Just as an illustration how widespread adoption of Semantic Web technology is becoming, here are just two startups I stumbled across before breakfast: Evri and InfoNgen. Both are tagging web-content, and serving the result back out again, InfoNgen as RDF, Evri in some proprietary XML format that could trivially be transformed to RDF (at least everything is URI-based). Evri also has a nice, simple visualisation, and some very good exposure through The Washington Post. And at least Evri is serving all this back out through an API for use by 3rd parties. Interesting option for a LarKC plugin!
