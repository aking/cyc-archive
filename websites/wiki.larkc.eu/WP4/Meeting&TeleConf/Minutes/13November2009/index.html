WP1+WP4 Joint Teleconference (13 November, 2009), 10:00-11:00

Participants

Barry (STI), Yi (WICI), Yan (WICI), Florian (STI), Gaston (VUA), Vassil (?OntoText), Zhisheng (VUA), Emanuele (CEFRIEL), Hans (MPG), Axel (HLRS).

Agenda

  • D1.1.4 Improved knowledge representation formalism WP1 leader: UIBK Month24
  • D1.2.2 Improved operational framework WP1 leader: VUA Month24
  • D4.3.2 Implemented plug-ins for interleaving reasoning and selection of axioms WP4 leader: VUA Month24

  • D4.4.1 Strategies & Design for rule-based inference WP4 leader: UIBK Month24

  • D4.6.1 Strategies & Design for stream reasoning WP4 leader: CEFRIEL Month24

D1.1.4

  • will add more content about complexity classes for different activities

D1.2.2

  • Gaston will organise a wp2 or 3 or 4 telco next week where we will brainstorm more re-use/closer integration possiblities. Aftert his first round we will build a matrix and then have another iteration going in to more depth. Possibly make use of the WP5 session in amsterdam to explore more possiblities face to face.

D4.3.2

D4.3.2 is a deliverable about the implementation of the reasoner plug-ins of interleaving reasoning and selection. It will consists of the following work:

  • i) (Amsterdam) the PION plug-in which is based on the PION system of reasoning with inconsistent ontologies. That would be developed based on the two different approaches: using selection functions which are implemented inside the plug-ins, and using the selection plug-in which are developed in WP2.
  • ii) (WICI) Interleaving reasoning and selection based on the granularity and multiple perspectives. The initial implementation of the starting point (one of the proposed strategy) plugin will be available before Month 24 (i.e., march 2010).
  • iii) (Amsterdam) Stefan's approach and Sypros' implementation (Gaston is going to add more here)
  • iv) (MPG and Amsterdam) (Hans is going to add more here).
  • v) (Stuttgart)Performance improvement by parallel reasoning. Stuttgart calls for the early release of the existing implementation of the reasoner plug-ins, so that they can be used for the analysis of the parallel strategies.

D4.4.1

Ontotext is interested in the development of fast consistency checker on large scale data. STI will cooperate with Ontotext to explore how the checker can be developed by using rule-based inference.

(Barry)

D4.6.1

The work of stream reasoning involve WP2, WP3, and WP4. The work of Deliverable D4.6.1 will focus on the perspective of reasoning in the context of WP4. Any partner who is interested in the contribution to the implementation of the WP4 stream reasoner pug-in is welcome.
