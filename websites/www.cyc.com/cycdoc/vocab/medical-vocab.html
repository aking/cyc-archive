#$PPTExplanation   p p t explanation    **COMMENT NOT REVIEWED**    **GAFs NOT REVIEWED**
PPT stands for ''#$PharmacyProductType''. A certain set of collections under #$PharmacyProductType exist chiefly to enforce the order of nesting of the functions that generate new #$PharmacyProductType instances by constraining their dosage form, administration route, package size, strength etc. These collection are: #$PPTByPackageType #$PPTByStrengthType #$PPTByAdministrationRouteType #$PPTByDosageFormType #$PharmacyProductType-Level3 #$PharmacyProductType-Level2 #$PharmacyProductType-Level1 #$PharmacyProductType-NameOnly. Also related are #$TherapeuticClassType and GlaxoProductGroupType-TopLevel. The hierarchy looks like this: #$PharmacyProductType | | | #$PPTByPackageType #$PharmacyProductType-Level3 | | | #$PPTByStrengthType #$PharmacyProductType-Level2 | | | #$PPTByAdministrationRouteType #$PharmacyProductType-Level1 | | | #$PPTByDosageFormType #$PharmacyProductType-NameOnly | | | #$TherapeuticClassType GlaxoProductGroupType-TopLevel The function #$PPTPackageFn, for example, results in an instance of #$PPTByPackageType, but it can take as arguments only #$PharmacyProductType-Level3 instances. #$PPTStrengthFn results in instances of #$PPTByStrengthType, but can take as arguments only instances of #$PharmacyProductType-Level2, and so on. This arrangement prevents an infinite recursion of NATs, e.g., (#$PPTPackageFn (#$PPTPackageFn (#$PPTPackageFn ... ))).
